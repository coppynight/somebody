# Voice Twin 产品方案（移动端原型版）

## 1. 产品定义
- 产品名：Voice Twin（语音数字人）
- 一句话：通过一次性的语音采访，完成人格建模与语音建模，生成可随时语音通话的“你的数字人”。
- 设计原则：全程以“打电话”体验为核心，减少文字和操作，强调陪伴感与仪式感。

## 2. 当前版本目标
- 产出可演示的手机 H5 交互原型，完整覆盖：
  1. 创建助理语音采访
  2. 建模仪式动画
  3. 数字人通讯录首页
  4. 发起并进行语音通话

## 3. 用户体验主线（一次性创建 + 长期使用）
1. 首次进入：与创建助理开始语音通话。
2. 采访采集：每次仅展示“当前问题”，底部仅“上一题/下一题”。
3. 建模阶段：进入仪式化动画，展示建模进度与关键步骤。
4. 创建完成：进入通讯录首页，置顶自己的数字人。
5. 日常使用：随时从通讯录发起语音电话。

## 4. 产品边界
- 采访与建模是一次性流程，不做多 Tab 复杂流程。
- 默认语音交互为主，文本仅用于必要提示（如实时转写、进度）。
- TELOS 十文件属于内部人格建模机制，不在前台向用户暴露。

## 5. 核心页面与功能
- `创建通话页`：头像动效 + 当前问题 + 精简进度 + 上一题/下一题。
- `建模仪式页`：环形/粒子感动效、步骤日志、百分比进度。
- `通讯录首页`：置顶“我的数字人”，支持对其他数字人拨号。
- `通话页`：电话式简洁布局（头像、计时、状态、实时转写、挂断）。
- `添加数字人`：通过数字 ID 添加。
- `设置`：查询、更新、分享自己的数字人。

## 6. 视觉与交互方向
- 风格关键词：科技感、高级感、沉浸式。
- 视觉手法：深色分层背景、霓虹线性高光、MBTI 动漫风 Avatar 动效。
- 交互节奏：少操作、强反馈、状态连续（聆听/思考/回答）。
